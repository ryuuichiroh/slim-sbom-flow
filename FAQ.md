# よくある質問と設計判断

以下は設計時に検討した項目と、その判断結果です。

## 脆弱性対応の記録更新に管理者の承認は不要か？

- 必要であれば、脆弱性検知時に GitHub Issue で「脆弱性対応」チケットを作成し、審査承認フローを経由して Dependency-Track を更新する運用にできます
- 当面は Dependency-Track の更新通知機能で対処します

## EOL を Dependency-Track / SBOM に含めるか？

- 利用している OSS の EOL 情報を記録したい場合に SBOM や DT のカスタムプロパティを利用することができます
- 定期的な EOL 通知で十分とし、カスタムプロパティへの EOL の記録は不要とします
- Xeol による EOL の検出が不足する場合は、代替手段をその時点で検討します

## SBOM を自動公開するか？

- SBOM は自動公開しません
- 外部提供が必要な場合は、申請を受けて審査し、承認後に Dependency-Track からエクスポートして提供します
